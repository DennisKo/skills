## Available Skills

### grug-brained-reviewer

Find architecture improvements before the clever architecture committee successfully summons the complexity demon and leaves future-you holding the pager. This skill walks the repo, sends focused explorer agents into the suspicious caves when available, connects actual evidence to grug-brained engineering patterns, and produces a self-contained HTML report of refactoring candidates that might reduce pain instead of merely redistributing it into folders with impressive names.

Inspired by and attributed to [The Grug Brained Developer](https://grugbrain.dev/), which says the quiet part about software complexity with more honesty than most architecture slide decks can survive.

**Use when:**

- Inspecting a project for architecture improvements before someone adds a platform layer to solve one boolean
- Looking for refactoring candidates that reduce complexity rather than laundering it through abstractions
- Producing an HTML architecture review report that names files, evidence, and pain instead of vibes
- Applying grug-brained engineering patterns to a codebase that has begun making developers whisper "why"
- Preparing to grill and harden a chosen refactoring candidate before implementation, because bravery is not a test plan
